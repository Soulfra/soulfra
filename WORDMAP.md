## Payment Concepts

### Bodega Receipt
**What:** Receipt-styled payment page (like a corner store receipt)
**Why:** Familiar aesthetic, trustworthy, nostalgic
**Tech:** Courier New font, CSS barcodes, perforated edges
**Example:** `pay-bodega.html`

### Reverse Receipt
**What:** Traditional flow is Pay â†’ Get Receipt. Reverse is: Scan Receipt â†’ Pay from Receipt
**Why:** Receipt IS the payment page
**Flow:**
1. Customer scans QR code
2. Opens receipt-styled page
3. Pays via Stripe/Venmo/etc from receipt
4. Receipt updates to show "PAID" stamp

### Payment QR
**What:** QR code that opens payment page
**Contains:** URL to `pay-bodega.html` with amount/item/stripe params
**Generated by:** `stpetepros-qr.html`

### Receipt QR
**What:** QR code given after payment
**Contains:** Link to paid receipt (proof of payment)
**Purpose:** Customer verification

## Payment Methods

### Stripe Payment Link
**What:** Stripe's hosted checkout page
**Format:** `https://buy.stripe.com/test_xxxxx`
**How we use it:** Embedded in iframe inside bodega receipt
**Advantage:** No backend needed, Stripe handles everything

### Deep Links
**What:** URL schemes that open mobile apps
**Examples:**
- Venmo: `venmo://pay?recipients=@user&amount=25`
- Cash App: `https://cash.app/$user/25`
- PayPal: `https://paypal.me/user/25`

**How they work:** Mobile browser â†’ Opens app â†’ Pre-filled amount

### Alternative Payment Methods
**What:** Non-Stripe payment options shown as fallback
**Options:** Venmo, Cash App, PayPal, Zelle
**Why:** Not everyone has Stripe, broader audience

## Technical Terms

### dist/
**What:** Distribution folder (standard in web dev)
**Contains:** Built/compiled files ready to deploy
**Used by:** React, Vue, Next.js, Vite, Webpack, etc.
**Why standard:** Industry convention - everyone knows what `dist/` means

### GitHub Pages
**What:** Free static site hosting by GitHub
**Serves from:** `dist/` folder (or any folder you specify)
**URL:** `https://username.github.io/repo-name/`
**Our URL:** `https://soulfra.github.io/bodega-demo.html`

### Static Site
**What:** Website with no backend/server
**Tech:** HTML, CSS, JavaScript only
**Advantages:**
- Fast (no database queries)
- Free hosting (GitHub Pages, Netlify, Vercel)
- Can't be hacked (no server to attack)
- Easy to deploy (just upload files)

**Our system:** 100% static - works on GitHub Pages

### Serverless
**What:** Backend code that runs on-demand, no server to manage
**Examples:** Cloudflare Workers, AWS Lambda, Vercel Functions
**Our backend:** Cloudflare Workers (`cloudflare-worker/payment-tracker.js`)

### Cloudflare Workers
**What:** JavaScript code that runs on Cloudflare's edge network
**Advantages:**
- Free tier: 100,000 requests/day
- No server to manage
- Runs globally (fast)
- Perfect for webhooks

**Our use case:** Track payments, handle Stripe webhooks

## QR Code Terms

### QR Code
**What:** Quick Response code - 2D barcode
**Contains:** URL, text, or data
**Scanned by:** Phone camera
**Library:** qrcode.js (CDN)

### QR Deep Link
**What:** QR code containing app deep link
**Example:** QR â†’ `venmo://pay?...` â†’ Opens Venmo app

### Dynamic QR
**What:** QR code that can be changed after printing
**How:** QR points to redirect URL, change redirect destination
**Not us:** Our QR codes are static (direct links)

### Error Correction
**What:** How much damage QR can take and still scan
**Levels:** L (7%), M (15%), Q (25%), H (30%)
**Our setting:** H (30%) - can work even if partially damaged

## Design Terms

### Receipt Aesthetic
**What:** Design that looks like a printed receipt
**Elements:**
- Courier New font (monospace)
- Black text on white
- Dashed borders
- Barcodes
- Perforated edges (radial gradients)
- Transaction IDs with letter-spacing

### Letterpress
**What:** Printing technique that leaves indented letters
**Digital version:** Drop shadows, emboss effects
**Our use:** "PAID" stamp has letterpress look

### Monospace Font
**What:** Font where all characters same width
**Examples:** Courier New, Consolas, Monaco
**Why:** Looks like typewriter/receipt printer

### Perforated Edge
**What:** Dotted line where paper tears
**CSS:** `radial-gradient(circle, transparent 40%, #fff 40%)`
**Our use:** Top/bottom of receipt

### Barcode Pattern
**What:** Black/white vertical lines
**CSS:** `repeating-linear-gradient(...)`
**Our use:** Separator between sections

## Data Terms

### localStorage
**What:** Browser storage that persists
**Capacity:** ~5-10MB
**Our use:** Save payment history, preferences
**Access:** `localStorage.setItem('key', 'value')`

### sessionStorage
**What:** Storage cleared when browser closes
**Our use:** Track if receipt was already shown (prevent re-print)

### KV Store
**What:** Key-Value store (simple database)
**Cloudflare KV:** Distributed storage for Workers
**Our KV namespaces:**
- `PAYMENTS` - Store payment records
- `RECEIPTS` - Store receipt data

### Transaction ID
**What:** Unique identifier for payment
**Our format:** `UPC-{timestamp}-{random}`
**Example:** `UPC-KL8X9P2Q-7DF3A`
