# Generated Templates Output

This directory contains code generated by the `templates_lib` modular template system.

## Directory Structure

- **database/** - SQL migrations, schemas, seed data, queries, triggers, views
- **python/** - Python CLI tools, async code, decorators, context managers
- **go/** - Go programs, servers, CLI tools
- **rust/** - Rust projects, crates, binaries
- **javascript/** - JavaScript/TypeScript code, React components, API clients
- **docker/** - Dockerfiles, docker-compose.yml, container configs
- **configs/** - Configuration files (tsconfig.json, pyproject.toml, Cargo.toml, etc.)
- **docs/** - READMEs, CONTRIBUTING guides, CHANGELOGs, ADRs
- **tests/** - Test templates (integration, E2E, performance, mocks)

## Usage

### Generate via CLI
```bash
# Generate a database migration
python3 template_generator.py --category database --name migration --params name=add_users

# Generate a Python CLI tool
python3 template_generator.py --category python --name cli-tool --params name=mytool
```

### Generate via Python API
```python
from templates_lib import generate_template, get_template

# Method 1: Direct generation
code = generate_template('database', 'migration', name='add_users')

# Method 2: Get template and generate
template = get_template('database', 'migration')
code = template.generate(name='add_users')

# Save to file
output_path = 'output/generated_templates/database/001_add_users.sql'
with open(output_path, 'w') as f:
    f.write(code)
```

### Search Templates
```python
from templates_lib import search_templates, list_templates, list_categories

# Search by keyword
results = search_templates('migration')
for template in results:
    print(f"{template.category}/{template.name}: {template.description}")

# List all categories
categories = list_categories()  # ['database', 'python', 'go', ...]

# List templates in category
templates = list_templates('database')  # {'database': ['migration', 'schema', ...]}
```

## File Naming Convention

Generated files follow these patterns:

- **Migrations**: `001_migration_name.sql`, `002_another_migration.sql`
- **Python**: `tool_name.py`, `module_name.py`
- **Go**: `main.go`, `server.go`, `cli.go`
- **Docker**: `Dockerfile`, `docker-compose.yml`
- **Configs**: `tsconfig.json`, `pyproject.toml`
- **Docs**: `README.md`, `CONTRIBUTING.md`

## Git Ignore

This directory is **ignored by git** (configured in `.gitignore`). Generated templates are treated as disposable outputs that can be recreated from the template library.

If you want to keep a generated file, move it to the appropriate location in your project:
```bash
# Example: Move generated migration to migrations/
mv output/generated_templates/database/001_add_users.sql migrations/

# Example: Move generated config to project root
mv output/generated_templates/configs/tsconfig.json .
```
