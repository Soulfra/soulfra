# Soulfra.com - Static Landing Page

**Purpose:** Public-facing marketing site with QR code for account creation

## What This Is

Static HTML landing page that:
- Introduces Soulfra concept
- Shows QR code for signup
- Links to blog posts
- Deploys to GitHub Pages (free)

## QR Code

The QR code on the landing page points to:
```
http://localhost:5002/qr-signup?ref=landing
```

Or in production:
```
https://soulfraapi.com/qr-signup?ref=landing
```

When user scans:
1. Opens soulfraapi.com/qr-signup
2. Creates account automatically
3. Redirects to soulfra.ai/?session=TOKEN
4. User can start chatting

## Files

- `index.html` - Main landing page
- `style.css` - Styling
- `qr-code.png` - QR code image (generated by ../generate-qr.py)
- `README.md` - This file

```bash

# Settings → Pages → Source: main branch
```

Live at: `https://<username>.github.io/soulfra-com`

## Custom Domain (Optional)

1. Buy domain: soulfra.com ($10-15/year)
2. Add CNAME record: `www.soulfra.com → <username>.github.io`
3. Add CNAME file:
   ```bash
   echo "soulfra.com" > CNAME
   git add CNAME && git commit -m "Add custom domain" && git push
   ```
4. GitHub Pages settings → Custom domain: soulfra.com
5. Wait 10-15 minutes for DNS propagation

## Update QR Code for Production

When deploying to production:

1. Update `../generate-qr.py` with production API URL:
   ```python
   API_URL = "https://soulfraapi.com/qr-signup?ref=landing"
   ```

2. Regenerate QR code:
   ```bash
   python3 ../generate-qr.py
   ```

3. Commit and push:
   ```bash
   git add qr-code.png
   git commit -m "Update QR code for production"
   git push
   ```
